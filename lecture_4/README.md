Student Grades Manager - Управление оценками студентов



Этот проект представляет собой систему управления базой данных для хранения информации о студентах и их оценках. Основная цель — продемонстрировать навыки работы с реляционными базами данных, SQL-запросами и оптимизацией производительности.



Структура проекта

create\_db.py — Создание и наполнение базы данных



Назначение: Автоматическое создание SQLite базы данных и заполнение её тестовыми данными.



Основная логика:



&nbsp;   Подключение к БД: Создаёт файл school.db в текущей директории



&nbsp;   Создание таблиц:



&nbsp;       students — хранит информацию о студентах (ID, ФИО, год рождения)



&nbsp;       grades — хранит оценки студентов по предметам с внешним ключом на таблицу студентов



&nbsp;   Вставка данных: Добавляет 9 студентов и по 3 оценки для каждого



&nbsp;   Оптимизация: Создаёт индексы для ускорения запросов по student\_id и subject



Ключевые особенности:



* &nbsp;   Использует sqlite3 модуль Python
* 
* &nbsp;   Имеет проверки целостности данных (CHECK constraint для оценок)
* 
* &nbsp;   Автоматически связывает оценки со студентами через внешние ключи
* 
* &nbsp;   Поддерживает повторный запуск (очищает старые данные)



Как использовать:



python create\_db.py



После выполнения появится файл school.db с готовой базой данных.

queries.sql — Аналитические SQL-запросы



Назначение: Содержит набор SQL-запросов для анализа данных о студентах и их оценках.



Содержание запросов:



* &nbsp;   Все оценки конкретного студента (Алисы Джонсон)
* 
* &nbsp;   Средний балл каждого студента с сортировкой по убыванию
* 
* &nbsp;   Студенты, родившиеся после 2004 года
* 
* &nbsp;   Средние оценки по всем предметам
* 
* &nbsp;   Топ-3 студента с наивысшим средним баллом
* 
* &nbsp;   Студенты с оценками ниже 80 баллов



Логика запросов:



&nbsp;   Использует JOIN для объединения таблиц



&nbsp;   Применяет агрегатные функции (AVG, COUNT)



&nbsp;   Использует фильтрацию (WHERE) и группировку (GROUP BY)



&nbsp;   Сортирует результаты (ORDER BY) и ограничивает вывод (LIMIT)



Как использовать:



sqlite3 school.db ".read queries.sql"



или в интерактивном режиме SQLite:



.read queries.sql



school.db — Файл базы данных SQLite



Назначение: Основной файл базы данных, содержащий все таблицы и данные.



Содержимое:



&nbsp;   Две таблицы: students (9 записей) и grades (27 записей)



&nbsp;   Созданные индексы для оптимизации



&nbsp;   Связи между таблицами через внешние ключи



Формат: Бинарный файл SQLite3, совместимый с любыми инструментами работы с SQLite.



Производительность



Проект включает оптимизации:



* &nbsp;   Индексы на часто запрашиваемых колонках
* 
* &nbsp;   Правильные типы данных для всех полей
* 
* &nbsp;   Ограничения целостности (CHECK, FOREIGN KEY)
* 
* &nbsp;   Эффективные JOIN запросы



Расширение функциональности



Чтобы добавить новые возможности:



* &nbsp;   Добавить новые таблицы: Измените create\_db.py
* 
* &nbsp;   Добавить новые запросы: Дополните queries.sql
* 
* &nbsp;   Добавить больше данных: Расширьте списки студентов и оценок в create\_db.py
* 
* &nbsp;   Добавить веб-интерфейс: Используйте Flask/Django с текущей базой



Примечания



* &nbsp;   Файл school.db создаётся автоматически при запуске create\_db.py
* 
* &nbsp;   При повторном запуске create\_db.py старые данные удаляются
* 
* &nbsp;   Все запросы в queries.sql являются SELECT-запросами и не изменяют данные
* 
* &nbsp;   Для изменения структуры базы потребуется редактирование create\_db.py
